# Atlas — Original brief

> Preserved verbatim for reference.

Private project + knowledge workspace for the user and their partner. The lovechild of Notion, Obsidian, Linear, and Apple Notes — designed in 2026.

## Four core surfaces

1. **Project boards** — Kanban-first, clean, visual, with list and possibly timeline views.
2. **Notes** — block-based, flexible, supports rich content, links to other notes.
3. **Brain dump inbox** — fast, almost-thoughtless capture surface for text, voice, images, files, links. Messy by design. Later organized into projects.
4. **Knowledge graph / backlinks** — every note references others; see how things connect.

## Feel

Luxurious. Calm. Confident. Nothing shouts; everything responds. 60fps everywhere. The app should feel like a very expensive, very well-made object — not a web app.

Aesthetic anchor: Apple circa 2026 — where iOS and macOS are heading. Spacious, generous whitespace, restrained color, deep attention to typography, soft depth through blur and subtle shadow rather than hard borders. Dark mode is first-class.

## Motion

Motion is the product. Spring-based, not tween-based. Drag has mass. Panels ease in. Hover states subtle but present. Page transitions continuous. Gesture-driven where it makes sense.

References: Linear (speed + precision), Things 3 (calmness + small delights), Arc (spatial, playful), Apple iOS transitions (gold standard for spring curves).

## Layout + responsiveness

Desktop-first but genuinely responsive. Mobile should feel designed, not crammed — capture and consumption dominate; organization secondary (bottom sheets, pull-to-refresh, swipe gestures). Desktop leans into keyboard-first power use (command palette like Raycast).

## Bilingual / RTL

Hebrew and English. Full RTL layout mirroring. Typography must work in both scripts.

## Key surfaces

- Project board (Kanban + list)
- Note editor (block-based)
- Brain dump inbox (capture + organize flow)
- Quick capture overlay (global shortcut + voice)
- Knowledge graph / backlinks panel
- Command palette (fuzzy + semantic)
- Settings + empty states

## Avoid

- Generic AI-app aesthetic (purple gradients, robotic feel, chat-bubble everything)
- Heavy borders and hard rectangles
- Information density for its own sake
- Trend skeuomorphism without purpose
- Inconsistent spacing or type scales

## Deliverables

1. Direction pass
2. Typography system, spacing scale, color tokens (light + dark)
3. Motion principles with specific spring values, durations, easing curves
4. Hi-fi designs for key surfaces, desktop + mobile
5. Prototype / motion spec for: card drag, quick-capture flow, page transition

> "3 screens that nail the feel" > "20 that approximate it."

## Additional

Clean design like Apple, but use color and vibrancy when it fits.
